01 — PROFILE

O que é um Perfil (PROFILE)?

📦 Definição
Conjunto nomeado de regras e limites para senha e uso de recursos. Atribuído a usuários na criação ou via ALTER USER.
✅ Pode
Estar ativado ou desativado. Ser atribuído a múltiplos usuários. Ser alterado a qualquer momento.
❌ Não pode
Ser atribuído a ROLES ou outros PROFILEs. Afetar sessões já ativas no momento da alteração.
⭐ PROFILE DEFAULT
Todo usuário criado sem especificar um profile recebe automaticamente o DEFAULT. Ele não pode ser eliminado, mas pode ser alterado.
📋 Dois grupos de parâmetros
Senhas — controle de complexidade, bloqueio, expiração e histórico
Recursos — CPU, I/O, sessões simultâneas, tempo de inatividade

02 — Senhas

Bloqueio de Conta & Expiração

🔒 Bloqueio de Conta

FAILED_LOGIN_ATTEMPTS
Número de tentativas de login malsucedidas antes de bloquear a conta. Ex: 3 tentativas.
PASSWORD_LOCK_TIME
Quantos dias a conta fica bloqueada após exceder as tentativas. Use 1/1440 para 1 minuto.

⏰ Expiração e Vencimento

PASSWORD_LIFE_TIME
Duração da senha em dias. Após esse período a senha expira. Ex: 30 dias.
PASSWORD_GRACE_TIME
Período de tolerância em dias para troca após o primeiro login com a senha expirada. Ex: 5 dias.
⏱️ Frações de dia: Para períodos menores que 1 dia, use frações. Ex: 1 hora = 1/24 | 5 minutos = 5/1440

02 — Senhas

Histórico & Verificação de Complexidade

🔁 Histórico de Senha

PASSWORD_REUSE_TIME
Número de dias que devem passar antes de reutilizar a mesma senha.
PASSWORD_REUSE_MAX
Número de vezes que a senha precisa ser trocada antes de poder reutilizá-la.

🔍 Verificação de Complexidade

PASSWORD_VERIFY_FUNCTION
Função PL/SQL que valida a senha. Pode ser a VERIFY_FUNCTION do Oracle ou uma criada pelo DBA.
Regras da VERIFY_FUNCTION (Oracle):
  • Mínimo de 4 caracteres
  • Não pode ser igual ao nome do usuário
  • Pelo menos 1 letra, 1 número e 1 especial
  • Pelo menos 3 letras diferentes da senha anterior
  • Criada pelo script utlpwdmg.sql

03 — Recursos

Limites de Recursos — Nível de Sessão

Aplicados a cada conexão. Ao exceder, o usuário é desconectado (erro ORA-02391).

SESSIONS_PER_USER
Número de sessões simultâneas permitidas para o usuário.
CONNECT_TIME
Tempo máximo de conexão em minutos.
IDLE_TIME
Período máximo de inatividade em minutos.
CPU_PER_SESSION
Tempo total de CPU em centésimos de segundo.
LOGICAL_READS_PER_SESSION
Número de blocos lidos (físicos + lógicos).
PRIVATE_SGA
Espaço privado na SGA em bytes (apenas Shared Server).
COMPOSITE_LIMIT: Limite ponderado que combina CONNECT_TIME + CPU_PER_SESSION + LOGICAL_READS + PRIVATE_SGA em "unidades de serviço". Modificado via ALTER RESOURCE_COST.

03 — Recursos

Limites por Chamada & Ativação

⚡ Nível de Chamada (por instrução SQL)

CPU_PER_CALL
Tempo de CPU por instrução em centésimos de segundo.
LOGICAL_READS_PER_CALL
Número de blocos lidos por instrução SQL.
⚠️ Ao exceder o limite por chamada:
  • A instrução é interrompida e revertida (ROLLBACK automático)
  • Instruções anteriores permanecem intactas
  • O usuário continua conectado

🔋 Ativando limites de recursos

Parâmetro RESOURCE_LIMIT
Os limites de recursos do PROFILE só são efetivos se o parâmetro estiver TRUE.
-- Ativar limites de recursos
ALTER SYSTEM SET RESOURCE_LIMIT = TRUE;

-- Desativar limites de recursos
ALTER SYSTEM SET RESOURCE_LIMIT = FALSE;

-- Válido até o banco ser desligado
Nota: Os limites de senha estão sempre ativos, independente do RESOURCE_LIMIT.

04 — Comandos

CREATE, ALTER e DROP PROFILE

CREATE PROFILE — Exemplo completo

CREATE PROFILE pro_gerente LIMIT
  -- Senha
  FAILED_LOGIN_ATTEMPTS 3
  PASSWORD_LOCK_TIME UNLIMITED
  PASSWORD_LIFE_TIME 30 -- dias
  PASSWORD_GRACE_TIME 5 -- dias
  PASSWORD_REUSE_TIME 30 -- dias
  PASSWORD_VERIFY_FUNCTION DEFAULT
  -- Recursos
  SESSIONS_PER_USER UNLIMITED
  CONNECT_TIME 45
  CPU_PER_CALL 3000
  LOGICAL_READS_PER_CALL 1000
  PRIVATE_SGA 15K
  COMPOSITE_LIMIT 5000000;

ALTER PROFILE — Modificar

-- Alterar parâmetros do DEFAULT
ALTER PROFILE DEFAULT LIMIT
  PASSWORD_LIFE_TIME 60
  PASSWORD_GRACE_TIME 10
  FAILED_LOGIN_ATTEMPTS 3
  PASSWORD_LOCK_TIME 1/1440; -- 1 min

DROP PROFILE — Excluir

-- Remover perfil (sem usuários)
DROP PROFILE pro_gerente;

-- Remove e re-atribui DEFAULT
-- aos usuários do perfil
DROP PROFILE pro_gerente CASCADE;

-- ⚠️ DEFAULT não pode ser removido!

04 — Referência

Atribuir Profile & Consultar Informações

Atribuir profile ao usuário

-- Na criação do usuário
CREATE USER joao
  IDENTIFIED BY senha123
  PROFILE pro_gerente;

-- Alterando perfil existente
ALTER USER joao
  PROFILE pro_gerente;

-- Voltar ao perfil padrão
ALTER USER joao
  PROFILE DEFAULT;
📊 Views de Consulta
DBA_PROFILES— todos os parâmetros dos perfis
DBA_USERS— profile atribuído a cada usuário
Limites de senha são sempre verificados pelo Oracle, independente do RESOURCE_LIMIT.
Recurso avançado: Para controle mais sofisticado de recursos, use o Database Resource Manager (DBMS_RESOURCE_MANAGER).
