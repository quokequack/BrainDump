Ativos, Ameaças e Riscos

Tipos de Ameaças

🌪️ Naturais — fogo, inundação, falta de energia, intempéries
🎯 Intencionais — fraudes, sabotagem, espionagem, invasões, furto de dados
😶 Involuntárias — erros de usuários, falta de conhecimento, descuido

Ativos e Vulnerabilidades

💻 Software — falhas de atualização, configuração, implementação
🖥️ Hardware — infraestrutura elétrica, inundações, falhas físicas
📁 Dados — controle de acesso deficiente, política de backup inadequada
👥 Pessoas — resistência às práticas de segurança, descuido
📌 Risco = probabilidade de que uma ameaça explore uma vulnerabilidade, causando danos. "A segurança deve ser proporcional ao valor do que se protege." — Thomas Wadlow

Métricas

Indicadores que guiam as soluções

MTBF Mean Time Between Failure
Tempo médio entre falhas. Quanto maior, mais confiável o sistema. Mede a frequência com que falhas ocorrem.
MTTR Mean Time To Recover
Tempo médio para recuperação após uma falha. Quanto menor, mais ágil a equipe. Mede a eficiência do processo de recuperação.
RTO Recovery Time Objective
Tempo máximo aceitável para recuperação após incidente. Ex: "O banco deve estar de volta em 4 horas". Define qual tecnologia usar (RMAN, Data Guard, RAC).
RPO Recovery Point Objective
Perda de dados máxima aceitável. Ex: "Não podemos perder mais de 1 hora de transações". Define frequência do backup e uso de archivelog.
💡 RTO pequeno → Data Guard / RAC (failover automático)  |  RPO pequeno → ARCHIVELOG + RMAN + Data Guard Sync

Tipos de Indisponibilidade e Soluções Oracle

⚡ Lentidão / Indisponibilidade Relativa
Usuário consome recursos excessivos
Solução: CREATE PROFILE com limites de CPU_PER_SESSION, SESSIONS_PER_USER, IDLE_TIME + Database Resource Manager (DBMS_RESOURCE_MANAGER)
🔴 Queda sem Avaria Permanente
Ex: falta de energia, crash de instância
Solução: Recuperação automática pelo processo SMON após STARTUP. Configurar FAST_START_MTTR_TARGET para limitar tempo de recovery.
🟠 Avaria de Software/Hardware
Servidor inutilizado temporariamente
Solução: Active Data Guard (Ativo/Passivo) — standby sempre pronto para assumir. ALTER DATABASE RECOVER MANAGED STANDBY DATABASE
💀 Perda Total do Data Center
Catástrofe — prédio, região
Solução: Oracle RAC (Ativo/Ativo) em site remoto + espelhamento ASM + RMAN com backups offsite. DATA GUARD FAR SYNC para RPO zero.

Confidencialidade

Criptografia — TDE e Network

🔐 TDE — Transparent Data Encryption
-- Criar wallet (keystore) ADMINISTER KEY MANAGEMENT CREATE KEYSTORE '/etc/oracle/wallet' IDENTIFIED BY "SenhaForte#1"; -- Abrir keystore ADMINISTER KEY MANAGEMENT SET KEYSTORE OPEN IDENTIFIED BY "SenhaForte#1"; -- Ativar chave master ADMINISTER KEY MANAGEMENT SET KEY IDENTIFIED BY "SenhaForte#1" WITH BACKUP; -- Criptografar coluna ALTER TABLE hr.employees MODIFY (salary ENCRYPT USING AES256); -- Criptografar tablespace ALTER TABLESPACE ts_dados ENCRYPTION ONLINE ENCRYPT;
🌐 Criptografia em Rede (sqlnet.ora)
-- sqlnet.ora (servidor) SQLNET.ENCRYPTION_SERVER = REQUIRED SQLNET.ENCRYPTION_TYPES_SERVER = (AES256, AES192, AES128) SQLNET.CRYPTO_CHECKSUM_SERVER = REQUIRED -- sqlnet.ora (cliente) SQLNET.ENCRYPTION_CLIENT = REQUESTED SQLNET.ENCRYPTION_TYPES_CLIENT = (AES256) -- Verificar sessão criptografada SELECT network_service_banner FROM v$session_connect_info WHERE sid = SYS_CONTEXT('USERENV','SID');
💡 TDE protege dados em repouso (arquivos, backups). sqlnet.ora protege dados em trânsito (rede).

Roteiros de Recovery — RMAN Passo a Passo

📋 ROTEIRO-01 — Restaurar da FRA
-- Identificar arquivo danificado SELECT file#, name, status FROM v$datafile WHERE status='RECOVER'; -- Conectar RMAN RMAN TARGET / -- Restaurar datafile específico RUN { SQL "ALTER DATABASE DATAFILE 5 OFFLINE"; RESTORE DATAFILE 5; RECOVER DATAFILE 5; SQL "ALTER DATABASE DATAFILE 5 ONLINE"; }
📋 ROTEIRO-02 — Restore do Backup em Disco
-- Especificar localização do backup RMAN> CATALOG START WITH '/u01/backup/'; -- Listar backups disponíveis RMAN> LIST BACKUP OF DATAFILE 5; -- Restaurar a partir do backup RUN { SET UNTIL SCN ; RESTORE DATAFILE 5; RECOVER DATAFILE 5; SQL "ALTER DATABASE DATAFILE 5 ONLINE"; } -- Verificar consistência RMAN> VALIDATE DATABASE;
⚠️ ROTEIRO-03 (Fita): mesma lógica do ROTEIRO-02, mas primeiro CATALOG aponta para o dispositivo de fita. Necessário ter RMAN configurado com CONFIGURE CHANNEL DEVICE TYPE SBT.
