Categorias de Falhas

⚡
Falha na Instância
Falta de energia, hardware, processos background. Recuperação automática via SMON.
👤
Erros do Usuário
DROP/TRUNCATE TABLE, UPDATE/DELETE sem WHERE correto. Usar Flashback, LogMiner ou Export.
💿
Perda de Mídia
Arquivo corrompido/perdido. Exige cópias físicas. Solução: RMAN, ASM, clusters.
🌪️
Catástrofes
Perda total do data center. Requer site backup, espelhamento remoto, RMAN + clusters.
💡 Cluster options: Ativo/Passivo (Active Data Guard) · Ativo/Ativo (Oracle RAC)

Flashback Technologies — Visão Geral

⚡ Flashback Drop
Fonte: Recycle Bin
Desfaz um DROP TABLE. Tabela vai para RECYCLEBIN com nome BIN$...
📋 Flashback Table
Fonte: Dados de Undo
Retorna tabela a SCN/TIMESTAMP anterior. Exige ENABLE ROW MOVEMENT.
🔎 Flashback Query
Fonte: Dados de Undo
Consulta dados de um ponto no passado via AS OF SCN ou AS OF TIMESTAMP.
🕐 Flashback Versions
Fonte: Dados de Undo
Vê todas as versões de linhas num intervalo via VERSIONS BETWEEN.
🔄 Flashback Transaction
Fonte: Dados de Undo
Reverte transação e dependentes via TRANSACTION_BACKOUT. Opções: CASCADE, NOCASCADE_FORCE.
🗄️ Flashback Database
Fonte: Flashback Logs
Reverte o banco inteiro. Requer modo ARCHIVELOG + FRA + ALTER DATABASE FLASHBACK ON.

Flashback Query & Table — Comandos

🔎 Flashback Query — por Sessão
-- Habilita toda a sessão num SCN EXECUTE DBMS_FLASHBACK .ENABLE_AT_SYSTEM_CHANGE_NUMBER(scn); -- ou por timestamp EXECUTE DBMS_FLASHBACK .ENABLE_AT_TIME(TO_TIMESTAMP(...)); -- Desabilita EXECUTE DBMS_FLASHBACK.DISABLE;
🔎 Flashback Query — por Comando
-- AS OF SCN SELECT * FROM emp AS OF SCN 1254890; -- AS OF TIMESTAMP (1h atrás) SELECT * FROM emp AS OF TIMESTAMP (SYSTIMESTAMP - INTERVAL '1' HOUR);
📋 Flashback Table
-- Habilitar Row Movement ALTER TABLE emp ENABLE ROW MOVEMENT; -- Retornar ao SCN FLASHBACK TABLE emp TO SCN 1254890; -- Retornar ao Timestamp FLASHBACK TABLE emp TO TIMESTAMP TO_TIMESTAMP('2025-04-16 14:00:00', 'YYYY-MM-DD HH24:MI:SS');
🗑️ Flashback Drop (Recyclebin)
-- Recuperar tabela da lixeira FLASHBACK TABLE emp TO BEFORE DROP [RENAME TO emp_restored]; -- Limpar lixeira PURGE TABLE emp; PURGE RECYCLEBIN; PURGE DBA_RECYCLEBIN; -- Ligar/desligar Recyclebin ALTER SESSION SET recyclebin = ON;

Flashback Data Archive — Histórico Permanente

⚠️ Flashback Table/Query usam UNDO (temporário). Flashback Data Archive guarda histórico por anos em tablespace dedicada.
📦 Criar e Habilitar
-- Criar archive com retenção 5 anos CREATE FLASHBACK ARCHIVE meu_arqfda TABLESPACE ts_flashback RETENTION 5 YEAR; -- Habilitar na tabela (usa DEFAULT) ALTER TABLE emp FLASHBACK ARCHIVE; -- Habilitar com archive específico ALTER TABLE emp FLASHBACK ARCHIVE meu_arqfda; -- Desabilitar ALTER TABLE emp NO FLASHBACK ARCHIVE;
🔍 Consultar e Gerenciar
-- Consultar histórico completo SELECT versions_starttime, versions_operation, emp.* FROM emp VERSIONS BETWEEN TIMESTAMP MINVALUE AND MAXVALUE; -- Views de monitoramento SELECT * FROM DBA_FLASHBACK_ARCHIVE; SELECT * FROM DBA_FLASHBACK_ARCHIVE_TABLES; SELECT * FROM DBA_FLASHBACK_ARCHIVE_TS; -- Excluir archive DROP FLASHBACK ARCHIVE meu_arqfda;

Data Pump — EXPDP / IMPDP

⚙️ Configuração
-- 1. Criar diretório Oracle CREATE DIRECTORY dpump_dir AS '/u01/backup'; -- 2. Conceder privilégios GRANT READ, WRITE ON DIRECTORY dpump_dir TO hr; -- 3. Grant de ROLEs especiais GRANT DATAPUMP_EXP_FULL_DATABASE TO hr; GRANT DATAPUMP_IMP_FULL_DATABASE TO hr;
▶️ Execução
-- Export completo expdp user/pass FULL=y DIRECTORY=dpump_dir DUMPFILE=full.dmp LOGFILE=exp.log -- Import completo impdp user/pass FULL=y DIRECTORY=dpump_dir DUMPFILE=full.dmp LOGFILE=imp.log -- Parâmetros avançados REMAP_SCHEMA=hr:scott QUERY=dept:"WHERE id < 120" EXCLUDE=PACKAGE CONTENT=DATA_ONLY

Backup Físico

RMAN — Configuração

⚙️ Configurar RMAN
-- Conectar RMAN TARGET SYS/pass@BANCO CATALOG rman/rman@BANCO -- Ver configurações RMAN> SHOW ALL; -- Destino dos backups CONFIGURE CHANNEL DEVICE TYPE DISK FORMAT 'c:\backup\%U'; -- Autobackup controlfile CONFIGURE CONTROLFILE AUTOBACKUP ON; CONFIGURE CONTROLFILE AUTOBACKUP FORMAT FOR DEVICE TYPE DISK TO FORMAT 'c:\backup\%F'; -- Política de retenção CONFIGURE RETENTION POLICY TO RECOVERY WINDOW OF 2 DAYS; -- ou por redundância CONFIGURE RETENTION POLICY TO REDUNDANCY 3;
▶️ Executar Backup
-- Full backup + archivelogs BACKUP DATABASE PLUS ARCHIVELOG; -- Só controlfile BACKUP CURRENT CONTROLFILE; -- Backup incremental nível 0 (base) BACKUP INCREMENTAL LEVEL 0 DATABASE PLUS ARCHIVELOG; -- Incremental diferencial nível 1 BACKUP INCREMENTAL LEVEL 1 DATABASE; -- Incremental cumulativo BACKUP INCREMENTAL LEVEL 1 CUMULATIVE DATABASE; -- Script complexo RUN { BACKUP DATABASE PLUS ARCHIVELOG; BACKUP CURRENT CONTROLFILE; };

RMAN — Restore & Recover

🔄 Recover Completo
RMAN> startup force nomount; -- Restaura controlfile do autobackup RMAN> RESTORE CONTROLFILE FROM AUTOBACKUP; RMAN> startup force mount; -- Restaura e recupera banco RMAN> RESTORE DATABASE; RMAN> RECOVER DATABASE; RMAN> ALTER DATABASE OPEN; -- Se recover incompleto: RMAN> ALTER DATABASE OPEN RESETLOGS;
⏱️ Recover Incompleto (PITR)
RUN { SET UNTIL SCN 1418901; RESTORE DATABASE; RECOVER DATABASE; } ALTER DATABASE OPEN RESETLOGS; -- Ou por data/hora RUN { SET UNTIL TIME "TO_DATE('23-DEC-2013 12:00:00', 'DD-MON-YYYY HH24:MI:SS')"; RESTORE DATABASE; RECOVER DATABASE; } ALTER DATABASE OPEN RESETLOGS;

RMAN com CDB e PDBs

💾 Backup CDB/PDB
-- Full CDB (inclui todos PDBs) RMAN> BACKUP DATABASE PLUS ARCHIVELOG; -- Só ROOT container RMAN> BACKUP DATABASE ROOT; -- PDB específico (conectado ao CDB) RMAN> BACKUP PLUGGABLE DATABASE pdb1, pdb2; -- Tablespace de PDB específico RMAN> BACKUP TABLESPACE pdb1:system, pdb1:users; -- Conectado diretamente no PDB RMAN> BACKUP DATABASE;
🔄 Recovery CDB/PDB
-- Recovery completo do CDB RUN { SHUTDOWN IMMEDIATE; STARTUP MOUNT; RESTORE DATABASE; RECOVER DATABASE; ALTER DATABASE OPEN; } -- Recovery de PDB específico RUN { ALTER PLUGGABLE DATABASE pdb1 CLOSE; RESTORE PLUGGABLE DATABASE pdb1; RECOVER PLUGGABLE DATABASE pdb1; ALTER PLUGGABLE DATABASE pdb1 OPEN; } -- PITR em PDB SET UNTIL TIME "TO_DATE(...)"; RESTORE PLUGGABLE DATABASE pdb1; RECOVER PLUGGABLE DATABASE pdb1; ALTER PLUGGABLE DATABASE pdb1 OPEN RESETLOGS;
