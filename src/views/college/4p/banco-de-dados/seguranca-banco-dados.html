01 — Usuários

CREATE USER — Criando um Usuário

CREATE USER joao
  IDENTIFIED BY senha123
  DEFAULT TABLESPACE users
  TEMPORARY TABLESPACE temp
  QUOTA 100M ON users
  ACCOUNT UNLOCK;

-- Forçar troca de senha no primeiro acesso
CREATE USER maria
  IDENTIFIED BY tmp
  PASSWORD EXPIRE
  ACCOUNT LOCK;
IDENTIFIED BY
Define a senha para autenticação via banco de dados. Alternativa: EXTERNALLY (SO) ou via rede (SSL, LDAP).
QUOTA {K|M|G} ON tablespace
Limita o espaço que o usuário pode usar no tablespace. UNLIMITED remove o limite — mas o privilégio UNLIMITED TABLESPACE tem prioridade.
PASSWORD EXPIRE / ACCOUNT LOCK
EXPIRE força troca na próxima conexão. LOCK bloqueia o acesso imediatamente.

01 — Usuários

ALTER USER & DROP USER

ALTER USER — Alteração

-- Trocar senha
ALTER USER joao
  IDENTIFIED BY novaSenha;

-- Alterar tablespace e quota
ALTER USER joao
  DEFAULT TABLESPACE dados
  QUOTA UNLIMITED ON dados;

-- Bloquear conta
ALTER USER joao ACCOUNT LOCK;

-- Definir tablespace padrão global
ALTER DATABASE DEFAULT TABLESPACE users;
ALTER DATABASE DEFAULT TEMPORARY TABLESPACE temp;

DROP USER — Exclusão

-- Remover usuário sem objetos
DROP USER joao;

-- Remover usuário E todos os objetos
-- do schema (tabelas, views, etc.)
DROP USER joao CASCADE;

-- ⚠️ Não é possível remover
-- usuário conectado no momento
⚠️ CASCADE é irreversível
Remove tabelas, views, índices e todos os objetos do schema do usuário. Não há como desfazer sem backup.

02 — Privilégios

GRANT — Concedendo Privilégios

De Sistema

-- Sintaxe
GRANT system_privilege
  TO user | role | PUBLIC
  [WITH ADMIN OPTION];

-- Exemplos
GRANT CREATE SESSION TO joao;
GRANT CREATE TABLE TO maria
  WITH ADMIN OPTION;
GRANT CREATE SESSION TO PUBLIC;

De Objetos

-- Sintaxe
GRANT object_privilege [(colunas)]
  | ALL [PRIVILEGES]
  ON objeto
  TO user | role | PUBLIC
  [WITH GRANT OPTION];

-- Exemplos reais
GRANT SELECT, INSERT, UPDATE, DELETE
  ON TAB_VEICULOS TO RO_FROTA;
GRANT EXECUTE ON PC_ESCALA TO RO_ESCALA;
GRANT UPDATE (STA_CONTRATO)
  ON TAB_CONTRATO TO RO_SUPERVISOR;
GRANT ALL ON TAB_CONVERSAO TO joao
  WITH GRANT OPTION;
WITH ADMIN OPTION (sistema) → permite ao recebedor conceder, revogar e até eliminar o privilégio para outros.
WITH GRANT OPTION (objetos) → permite ao recebedor conceder o privilégio a outros usuários.

02 — Privilégios

REVOKE — Revogando Privilégios

De Sistema

REVOKE system_privilege
  FROM user | role | PUBLIC;

-- Qualquer admin com ADMIN OPTION
-- pode revogar de qualquer usuário,
-- mesmo sem ter concedido antes

De Objetos

REVOKE object_privilege | ALL [PRIVILEGES]
  ON objeto
  FROM user | role | PUBLIC
  [CASCADE CONSTRAINTS];

-- CASCADE CONSTRAINTS remove restrições
-- de integridade referencial criadas
-- pelo revogado via REFERENCES ou ALL
Sistema: Quem tem ADMIN OPTION pode revogar de qualquer usuário.
Objetos: Somente quem concedeu o privilégio pode revogá-lo.

03 — Roles

CREATE ROLE & Gerenciamento

Criar e Alterar

-- Role sem senha
CREATE ROLE ro_vendedor
  NOT IDENTIFIED;

-- Role com senha (mais segura)
CREATE ROLE ro_supervisor
  IDENTIFIED BY segredo;

-- Alterar autenticação da role
ALTER ROLE ro_supervisor
  NOT IDENTIFIED;

-- Eliminar role (revoga de todos)
DROP ROLE ro_vendedor;

Conceder e Revogar Roles

-- Conceder role a usuário
GRANT ro_vendedor TO joao;
GRANT ro_supervisor TO maria
  WITH ADMIN OPTION;

-- Atribuir privilégios à role
GRANT SELECT ON TAB_CLIENTE
  TO ro_vendedor;

-- Revogar role do usuário
REVOKE ro_vendedor FROM joao;
WITH ADMIN OPTION em roles permite ao recebedor também conceder, revogar, alterar ou eliminar a role.

03 — Roles

Ativação e Desativação de Roles

ALTER USER — Roles padrão no logon

-- Nenhuma role ativa ao conectar
ALTER USER joao
  DEFAULT ROLE NONE;

-- Ativar somente uma role
ALTER USER joao
  DEFAULT ROLE ro_gerente;

-- Todas exceto uma
ALTER USER joao
  DEFAULT ROLE ALL EXCEPT ro_supervisor;

-- Restaurar todas
ALTER USER joao DEFAULT ROLE ALL;

SET ROLE — Durante a sessão

-- Desativar todas as roles agora
SET ROLE NONE;

-- Ativar role com senha
SET ROLE ro_gerente
  IDENTIFIED BY teste;

-- Ativar todas as roles concedidas
SET ROLE ALL;
-- (só funciona se nenhuma tem senha)

-- Todas exceto uma
SET ROLE ALL EXCEPT ro_supervisor;
Dica: DBMS_SESSION.SET_ROLE também pode ativar/desativar roles via PL/SQL dentro de procedures.
